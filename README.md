# supervised_phenotype_scoring

A local web application for anonymised, supervised scoring of birdsong
phenotype traits by human raters.

Songs are extracted from audio recordings, stored in an HDF5 batch file,
served via a drag-and-drop ranking interface with interactive spectrogram
comparison panels, and aggregated into per-bird Elo scores with inter-rater
reliability statistics.

Designed for cross-foster experiments where scorer identity knowledge would
bias trait ratings (stereotypy, repeat propensity, etc.).

**API docs:** https://annie-taylor.github.io/supervised_phenotype_scoring/

## Installation

### Requirements

- [Miniconda](https://docs.conda.io/en/latest/miniconda.html) or Anaconda
- Git

### Create the conda environment

```bash
git clone <repo-url> supervised_phenotype_scoring
cd supervised_phenotype_scoring
conda env create -f environment.yml
conda activate supervised_phenotype_scoring
```

This installs Python 3.11 plus all required packages.
`.cbin` audio support is provided by `tools/evfuncs.py` (a local copy
included in the repo) — no separate install needed.  The `boto3` package is
only needed for S3 upload (`upload_batch.py`).

### Verify the install

```bash
python make_test_batch.py
python export_batch.py batches/test_batch
python analyze_rankings.py batches/test_batch
```

You should see spectrogram PNGs appear in `batches/test_batch/export/spectrograms/`
and a summary printed to the console.

## Data preparation

### What you need

| File | Location | Description |
|------|----------|-------------|
| `nest_gen_pair_offspring_summary.csv` | `refs/` | One row per nest-father × genetic-father pairing with XF bird lists |
| `genetic_father_offspring_summary.csv` | `refs/` | Hand-reared bird lists per genetic father |
| `audio_candidates_cache.json` | `refs/` | `{bird_id: [{filepath, size_mb}, ...]}` — pre-screened audio paths per bird |

These files are generated by the upstream `xfoster_bird_selection` pipeline
(`file_management/identifying_families.ipynb`) and should be copied into the
`refs/` folder of this repo.  They are excluded from git (see `.gitignore`).

### config.json

```json
{
  "project_dir":            "E:/scoring/",
  "ref_dir":                "E:/scoring/refs/",
  "audio_candidates_cache": "E:/scoring/refs/audio_candidates_cache.json",
  "output_dir":             "E:/scoring/batches",

  "snippets_per_bird":  6,
  "snippet_duration_s": 8.0,
  "min_gap_s":          5.0,
  "edge_s":             1.0,
  "seed":               42,
  "target_sr":          32000,

  "spectrogram": {
    "nfft":     512,
    "hop":      1,
    "min_freq": 400,
    "max_freq": 10000,
    "p_low":    2,
    "p_high":   98
  },

  "n_workers": 4
}
```

**Key config fields:**

| Key | Description |
|-----|-------------|
| `ref_dir` | Directory containing the bird registry CSVs |
| `audio_candidates_cache` | JSON mapping bird IDs to candidate audio paths |
| `target_sr` | Target sample rate; source files at any rate are resampled automatically |
| `spectrogram.hop` | STFT hop in samples; `hop=1` stores maximum time-resolution spectrograms in HDF5 |
| `spectrogram.nfft` | FFT size (512 → 62.5 Hz/bin at 32 kHz) |

## Pipeline overview

```
run_pipeline.py --phase 1     prepare + export all pairings in one command
      ↓
prepare_batch.py              audio → HDF5 batch (spectrograms + audio + manifest)
      ↓
export_batch.py               HDF5 → PNG spectrograms + WAV audio clips
      ↓
prescreen_app.py              interactive QC: label each spectrogram song / not-song / error
      ↓
run_pipeline.py --phase 2     rebuild with exclusions + re-export all pairings
      ↓
ranking_app.py                Flask drag-and-drop scoring interface
      ↓
analyze_rankings.py           session JSON files → Elo scores + IRR + flagged-snippets CSV
```

Optionally, `printout_generator.py` produces a self-contained HTML file
suitable for printing and hand-scoring, and `upload_batch.py` pushes
exported assets to S3 for remote scoring sessions.

## Step-by-step usage

### Multi-family workflow (recommended)

Create a pairings CSV listing all nest-father × genetic-father combinations:

```
nest_father,genetic_father
pk24bu3,wh88br85
ab12cd3,ef45gh6
```

**Phase 1 — build and export all batches:**
```bash
python run_pipeline.py pairings.csv --phase 1
```

**Prescreen each batch** (see [Prescreen app](#prescreen-app) below):
```bash
python prescreen_app.py batches/pk24bu3_wh88br85_20260414
```

**Phase 2 — rebuild with exclusions and re-export:**
```bash
python run_pipeline.py pairings.csv --phase 2
```

Phase 2 automatically finds the prescreen CSV for each pairing, rebuilds
the batch excluding non-song snippets, and prints `scp` upload commands.

---

### Single-family workflow

#### 1. Build the batch

```bash
python prepare_batch.py --nest-father pk24bu3 --genetic-father wh88br85
```

This writes `batches/<nf>_<gf>_<YYYYMMDD>/batch.h5`.

Options:
- `--snippets-per-bird N` — override config (default 6)
- `--workers N` — parallel spectrogram workers (default 4)
- `--exclude-csv path/to/prescreen.csv` — skip non-song snippets from a prior prescreen

#### 2. Export spectrograms and audio

```bash
python export_batch.py batches/pk24bu3_wh88br85_20260414
```

Writes PNG and WAV files to `batches/.../export/`.

Options:
- `--dpi 150` — higher-resolution PNGs
- `--no-audio` — skip WAV export
- `--force` — overwrite existing files
- `--workers N`

#### 3. Prescreen the batch

```bash
python prescreen_app.py batches/pk24bu3_wh88br85_20260414
```

Open `http://localhost:5001/`.  For each spectrogram press **S** (song),
**N** (not song), or **E** (rendering error).  Labels are saved to
`batches/.../prescreen_<date>.csv`.  Resume at any time by restarting.

#### 4. Rebuild excluding non-song snippets, topping up where possible

```bash
python prepare_batch.py --nest-father pk24bu3 --genetic-father wh88br85 \
    --exclude-csv batches/pk24bu3_wh88br85_20260414/prescreen_20260414.csv \
    --existing-batch batches/pk24bu3_wh88br85_20260414/batch.h5
python export_batch.py batches/pk24bu3_wh88br85_20260414 --force
```

`--existing-batch` carries over valid snippets from the Phase 1 HDF5 without
recomputing them, and samples new positions only for birds below the target
count.  `run_pipeline.py --phase 2` passes this flag automatically.

#### 5. Run the scoring app

```bash
python ranking_app.py batches/pk24bu3_wh88br85_20260414
```

Open `http://localhost:5000/` in a browser.  Enter your name, choose a
trait, and drag song cards into ranked order (leftmost = most of trait).
Rankings are saved after every round to `batches/.../sessions/`.

On first run, `Sortable.min.js` and `plotly.min.js` are automatically
downloaded to `static/` if not already present (requires internet).

Options:
- `--port 5001`
- `--batch-size 7` — songs per ranking round

#### 6. Generate the print-scoring sheet (optional)

```bash
python printout_generator.py batches/pk24bu3_wh88br85_20260414
```

Produces `batches/.../printout_<trait>.html` — use File → Print in the
browser for a PDF.  Bird identities are hidden; an answer key is the last page.

#### 7. Analyze rankings

```bash
python analyze_rankings.py batches/pk24bu3_wh88br85_20260414 --trait all
```

Writes to `results/`:

| File | Description |
|------|-------------|
| `<batch>_<trait>_elo.csv` | Per-snippet Elo scores |
| `<batch>_<trait>_birds.csv` | Per-bird averages by role |
| `<batch>_<trait>_irr.csv` | Pairwise Kendall τ between scorers |
| `<batch>_<trait>_summary.txt` | Human-readable digest |
| `<batch>_flagged.csv` | Snippets flagged as noise/call during scoring |

## Prescreen app

The prescreen app lets you review every spectrogram before expert scorers see
the batch, filtering out noise, calls, and rendering errors.

```bash
python prescreen_app.py batches/<batch_name>
# open http://localhost:5001
```

Each spectrogram is shown as a full-resolution interactive Plotly heatmap
(magma colorscale, scroll to zoom, drag to pan).  Press:

| Key | Label | Effect |
|-----|-------|--------|
| S | song | Keep in batch |
| N | not song | Exclude — noise, calls, non-song vocalisations |
| E | rendering error | Exclude — display / normalisation artefact |

Labels are saved to `batches/.../prescreen_<date>.csv` after each decision.
Close and restart to resume mid-session.

The prescreen CSV is consumed by `prepare_batch.py --exclude-csv` in Phase 2
to rebuild the batch without the flagged snippets.

## Scoring app

### How scoring works

Each round presents `--batch-size` song cards (default 7).  Cards can be
dragged into order and reordered freely before submitting.  Click a
spectrogram image to play its audio clip.

The progress bar tracks songs ranked so far in the session.  Sessions can
be stopped at any time; progress is saved after every submitted round.

### Flagging noise and calls

Each card has a **flag noise/call** button.  Flagged cards are shown with a
red border and a ✕ rank badge and are excluded from the ranking on submit.

- Flagged UIDs are stored in the `flagged` list of each round record.
- `analyze_rankings.py` excludes any flagged UID from Elo initialisation
  and from all pairwise comparisons, so they cannot influence other scores.
- `<batch>_flagged.csv` records the source file and snippet start time of
  every flagged snippet for exclusion from future batches.

A snippet flagged by **any** scorer is excluded — expert scorers are assumed
to reliably identify noise and calls.

### Interactive comparison panels

Below the ranking row, two side-by-side **comparison panels** allow detailed
inspection before committing to a ranking:

1. Click **A** or **B** on any card to load that song into the corresponding panel.
2. Each panel renders a Plotly heatmap (time × frequency).
3. Scroll to zoom, click-drag to pan, double-click to reset view.
4. Load two songs into A and B to compare them at any zoom level.

Comparison panels fetch data on demand (`nfft=512, hop=64` from the exported
WAV, independent of the `hop=1` used for HDF5 storage) — each request is
roughly 3–4 MB.

### Anonymisation

Song cards show only an 8-character UID prefix.  UIDs are UUID4 strings
generated with a random per-batch salt, so the same song in two batches gets
different UIDs.

The uid → bird identity mapping exists only in `batch.h5` and in the answer
key page of `printout_<trait>.html`.

### Session files

Saved to `batches/<batch>/sessions/<scorer>_<trait>_<date>.json`:

```json
{
  "scorer":   "Annie",
  "trait":    "stereotypy",
  "platform": "local",
  "rounds": [
    {
      "round":     1,
      "presented": ["<uid>", ...],
      "ranking":   ["<uid>", ...],
      "flagged":   ["<uid>", ...],
      "elapsed_s": 42.1,
      "timestamp": "2026-04-10T14:32:01"
    }
  ]
}
```

`flagged` is an empty list when no snippets were flagged in that round.
This format is MTurk-compatible so local and crowdsourced sessions can be
pooled in `analyze_rankings.py`.

## Analysis

### Elo scoring

Each ranking is decomposed into all implied pairwise comparisons
(rank 1 beats rank 2, …, rank n−1 beats rank n).  Elo ratings are updated
per comparison (K = 32, base = 400, start = 1500).

Flagged snippets are excluded before initialisation and never participate
in any pairwise comparison, so they cannot affect other snippets' ratings.

Higher Elo = more of the trait.

### Noise / call exclusion output

`<batch>_flagged.csv` columns:

| Column | Description |
|--------|-------------|
| `uid` | Snippet UUID |
| `bird_id` | Internal bird identifier |
| `role` | `xf`, `hr_nest`, `nest_father`, etc. |
| `source_file` | Original audio file path |
| `snippet_start_s` | Start time within the source file |
| `n_scorers_flagged` | Number of scorers who flagged this snippet |
| `flagged_by` | Semicolon-separated scorer names |

### Inter-rater reliability

Kendall's τ-b is computed between every pair of scorers on their shared
songs.  τ ≈ 1 = perfect agreement; τ ≈ 0 = no agreement.

Rounds completed in under 5 seconds are flagged as suspicious and excluded.

### Role-level comparison

Mean Elo by role is reported, plus the XF − nest-father and XF − genetic-father
differences, making it easy to assess which tutor the cross-foster offspring
song is closer to.

## Hosted / MTurk mode

```bash
# 1. Upload assets to S3
python upload_batch.py batches/<batch> --bucket my-bucket --prefix scoring/<batch>

# 2. Run in hosted mode
python ranking_app.py batches/<batch> --mode hosted
```

`upload_batch.py` writes `config_hosted.json` with S3 base URLs after a
successful upload.  Use `--dry-run` to preview.
